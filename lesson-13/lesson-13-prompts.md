# 13차시: 자동매매 전략 최적화

## 1번 프롬프트

```text
자동매매 전략을 최적화하는 시스템을 설계해줘.
[최적화 목표]
- 수익률 최대화, 리스크 최소화
- 거래 비용 최적화, 안정성 향상
[현재 성능 지표]
- 월간 수익률: 5%, 최대 낙폭: 15%
- 승률: 60%, 샤프 비율: 1.2
[개선 목표]
- 월간 수익률: 8% 이상, 최대 낙폭: 10% 이하
- 승률: 70% 이상, 샤프 비율: 1.5 이상
[제약 조건]
- 최대 투자 금액: 1000만원, 일일 거래 횟수 제한: 20회
- 수수료: 0.05%, 거래 시간: 09:00-23:00
[평가 기준]
- 백테스트 결과, 실시간 성능
- 리스크 조정 수익률, 안정성 지표
이 요구사항을 만족하는 전략 최적화 시스템을 설계해줘.
```

## 2번 프롬프트

```text
최적화 접근법:
1. 파라미터 최적화
   - 변동성 돌파 전략의 k값 최적화
   - 이동평균 기간 최적화
   - 손절/익절 비율 최적화
2. 멀티 전략 최적화
   - 전략 조합 최적화
   - 가중치 할당 최적화
   - 동적 전략 전환
3. 시장 상황별 최적화
   - 변동성 구간별 최적화
   - 트렌드/사이드웨이스 최적화
   - 시간대별 최적화
4. 리스크 관리 최적화
   - 포지션 사이징 최적화
   - 손실 한계 최적화
   - 상관관계 분석
각 방법의 구현 방법과 예상 효과를 설명해줘.
```

## 3번 프롬프트

```text
위에서 제안한 최적화 방법들을 구현해줘
구현 요구사항:
1. 파라미터 최적화 엔진
2. 멀티 전략 관리 시스템
3. 시장 상황 분석 모듈
4. 리스크 관리 최적화
5. 성능 평가 시스템

각 최적화 방법이 실제로 성능을 향상시킬 수 있도록 구체적으로 구현해줘.
```

## 4번 프롬프트

```text
구현된 최적화 방법들의 효과를 검증해줘.

검증 방법:
1. 백테스트를 통한 성능 비교
2. 실시간 테스트
3. 통계적 유의성 검정
4. 리스크 조정 성과 분석
5. 안정성 테스트

각 최적화 방법의 효과를 정량적으로 평가하고, 실제 적용 가능성을 판단해줘
```

## 5번 프롬프트

```text
구현 요구사항:
1. 실시간 데이터 수집기
2. 성능 지표 계산 엔진
3. 알림 시스템
4. 대시보드
5. 데이터 저장 및 분석

시스템이 실시간으로 성능을 추적하고,
문제 상황을 즉시 감지할 수 있도록 구현해줘.
```

## 6번 프롬프트

```text
구현된 모니터링 시스템을 최적화 해줘.

최적화 포인트:
1. 데이터 수집 효율성
2. 실시간 처리 속도
3. 알림 정확성
4. 대시보드 사용성
5. 시스템 리소스 사용량

모니터링 시스템 자체가 성능에 영향을 주지 않으면서
정확한 정보를 제공하도록 최적화 해줘.
```

## 7번 프롬프트

```text
자동매매 시스템의 성능 병목을 분석하고 개선 방안을 제시해줘.
분석 영역:
1. 코드 성능
   - 알고리즘 복잡도, 메모리 사용 패턴
   - CPU 사용률, I/O 병목
2. 시스템 아키텍처
   - 모듈 간 통신, 데이터 흐름
   - 병렬 처리, 캐싱 전략
3. 외부 의존성
   - API 호출 최적화, 네트워크 지연
   - 데이터베이스 성능, 외부 서비스 응답
4. 리소스 관리
   - 메모리 누수, 연결 풀 관리
   - 파일 I/O 최적화
```

## 8번 프롬프트

```text
위에서 분석한 성능 병목을 개선해줘.
개선 요구사항:
1. 알고리즘 최적화
2. 메모리 사용량 최적화
3. API 호출 최적화
4. 데이터베이스 쿼리 최적화
5. 병렬 처리 구현
시스템의 전반적인 성능이 향상되도록 구체적인 개선 사항을 구현해줘.
```

## 9번 프롬프트

```text
성능 개선의 효과를 측정하고 분석해줘.
측정 지표:
1. 처리 속도 개선도
2. 메모리 사용량 감소율
3. CPU 사용률 개선
4. 응답 시간 단축
5. 전체 시스템 성능 향상
개선 전후를 비교하여 정량적인 효과를 측정하고,
추가 개선이 필요한 부분을 식별해줘.
```
