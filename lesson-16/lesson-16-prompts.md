# 16차시: MCP(Model Context Protocol) 활용

## 1번 프롬프트

```text
MCP(Model Context Protocol)의 개념과 활용법을 설명해줘.

1. MCP란 무엇인가요?
2. MCP의 주요 장점은 무엇인가요?
3. 어떤 도구들이 MCP를 지원하나요?
4. MCP를 활용한 개발 워크플로우는 어떻게 되나요?
5. MCP와 기존 API의 차이점은 무엇인가요?

자동매매 시스템 개발에서 MCP를 어떻게 활용할 수 있는지
구체적인 예시와 함께 설명해줘.
```

## 2번 프롬프트

```text
자동매매 시스템 개발을 위한 MCP 도구 연동 방법을 제안해줘.

활용할 MCP 도구:
1. Figma MCP (UI/UX 디자인)
2. Notion MCP (문서 관리)
3. Selenium MCP (웹 자동화)
4. Browser MCP (브라우저 제어)

각 도구별로:
- 주요 기능, 활용 시나리오
- 연동 방법, 예상 효과

설명하고, 자동매매 시스템 개발에 어떻게 활용 할 수 있는지 구체적으로 제시해줘.
```

## 3번 프롬프트

```text
Figma MCP를 활용하여 자동매매 시스템의 UI/UX 디자인을 생성해줘.

디자인 요구사항:
1. 대시보드 페이지
   - 실시간 수익률 표시
   - 거래 내역 테이블
   - 차트 및 그래프
   - 알림 섹션
2. 설정 페이지
   - 전략 설정 폼
   - API 키 관리
   - 알림 설정
   - 사용자 프로필
3. 모바일 반응형 디자인
   - 터치 친화적 인터페이스
   - 간소화된 네비게이션
   - 핵심 기능 우선 배치
4. 디자인 시스템
   - 일관된 색상 팔레트
   - 타이포그래피 규칙
   - 컴포넌트 라이브러리
   - 아이콘 세트

Figma MCP를 사용하여 이 요구사항을 만족하는 프로토타입을 생성해줘.
```

## 4번 프롬프트

```text
생성된 Figma 디자인의 사용자 경험을 최적화해줘.

최적화 포인트:
1. 사용성 개선
   - 직관적인 정보 구조
   - 효율적인 작업 플로우
   - 접근성 고려
2. 시각적 개선
   - 일관된 디자인 언어
   - 시각적 계층 구조
   - 브랜드 아이덴티티 반영
3. 기능적 개선
   - 핵심 기능 강조
   - 오류 상태 처리
   - 로딩 상태 표시
4. 반응형 최적화
   - 다양한 화면 크기 대응
   - 터치 인터페이스 최적화
   - 성능 고려

사용자가 쉽고 효율적으로 시스템을 사용할 수 있도록 디자인을 개선해줘.
```

## 5번 프롬프트

```text
Figma 디자인을 실제 웹 코드로 변환해줘.

변환 요구사항:
1. HTML 구조
   - 시맨틱 마크업
   - 접근성 고려
   - SEO 최적화
2. CSS 스타일링
   - 반응형 디자인
   - CSS Grid/Flexbox 활용
   - 애니메이션 효과
3. JavaScript 기능
   - 인터랙티브 요소
   - 데이터 바인딩
   - API 연동
4. 프레임워크 활용
   - React/Vue.js 컴포넌트
   - 상태 관리
   - 라우팅

Figma 디자인을 정확하게 구현하는 실행 가능한 웹 코드를 생성해줘.
```

## 6번 프롬프트

```text
자동매매 시스템을 위한 대시보드를 설계해줘

대시보드 구성:
1. 실시간 모니터링 섹션
   - 현재 포지션 상태
   - 실시간 수익률
   - 거래 활동 로그
   - 시스템 상태
2. 성과 분석 섹션
   - 수익률 차트
   - 거래 통계
   - 리스크 지표
   - 성과 비교
3. 설정 및 제어 섹션
   - 전략 설정
   - 거래 중지/시작
   - 알림 설정
   - 시스템 설정
4. 알림 및 메시지 섹션
   - 실시간 알림
   - 중요 메시지
   - 오류 경고
   - 성공 메시지

각 섹션의 레이아웃과 기능을 설계하고, 사용자 경험을 최적화해줘.
```

## 7번 프롬프트

```text
위에서 설계한 대시보드를 실제로 구현해줘.

구현 요구사항:
1. React 기반 컴포넌트 구조
2. 실시간 데이터 연동
3. 차트 라이브러리 활용 (Chart.js/Recharts)
4. 반응형 레이아웃
5. 다크/라이트 모드 지원

대시보드가 실시간으로 데이터를 표시하고,
사용자가 직관적으로 시스템을 제어할 수 있도록 구현해줘.
```
