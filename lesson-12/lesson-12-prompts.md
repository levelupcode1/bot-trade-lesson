# 12차시: 자동매매 시스템 거래 데이터 분석

## 1번 프롬프트

```text
자동매매 시스템의 거래 데이터를 분석하는 시스템을 설계해줘.
[분석 목적]
- 거래 성과 평가 및 개선
- 전략 최적화, 리스크 관리, 수익성 분석
[데이터 소스]
- 업비트 API 거래 내역, 가격 데이터 (OHLCV)
- 거래 로그, 시스템 상태 데이터
[분석 지표]
1. 수익률 지표
   - 총 수익률, 일간 수익률, 월간 수익률
   - 최대 낙폭 (MDD), 샤프 비율, 소르티노 비율
2. 거래 지표
   - 거래 횟수, 승률
   - 평균 보유 기간, 수수료 비용
3. 리스크 지표
   - 변동성, VaR (Value at Risk), 최대 손실
[시각화 요구사항]
- 수익률 차트 (누적, 일간), 거래 분포 히스토그램
- 월별 성과 히트맵, 리스크 - 수익 스캐터 플롯
[인사이트 요구사항]
- 성과 개선 포인트 식별, 최적 거래 시간대 분석
- 코인별 성과 비교, 전략별 성과 분석
```

## 2번 프롬프트

```text
자동매매 데이터를 분석하는 Python 코드를 작성해줘.
요구사항:
1. pandas, numpy를 활용한 데이터 처리
2. matplotlib, seaborn을 활용한 시각화
3. scipy를 활용한 통계 분석
4. 모듈화된 함수 구조
5. 설정 가능한 파라미터
분석 기능:
- 거래 데이터 로드 및 전처리
- 수익률 계산 및 분석, 리스크 지표 계산
- 시각화 생성, 리포트 생성
확장 가능하고 재사용 가능한 코드 구조로 작성해줘.
```

## 3번 프롬프트

```text
구현된 데이터 분석 코드를 최적화 해줘.
최적화 포인트:
1. 처리 속도 개선
2. 메모리 사용량 최적화
3. 코드 가독성 향상
4. 오류 처리 강화
5. 확장성 개선
대용량 데이터도 효율적으로 처리할 수 있도록 성능을 최적화 해줘.
```

## 4번 프롬프트

```text
자동매매 시스템을 위한 자동 리포트 생성 시스템을 설계해줘.
리포트 유형:
1. 일간 리포트 (거래 요약, 수익률, 주요 이벤트)
2. 주간 리포트 (성과 분석, 전략 평가, 개선점)
3. 월간 리포트 (종합 성과, 시장 분석, 전략 조정)
4. 특별 리포트 (이상 상황, 긴급 알림)
리포트 구성:
- 실행 요약, 성과 지표, 시각화 차트
- 인사이트 및 권장사항, 향후 계획
자동 생성 요구사항:
- 정기 스케줄링, 템플릿 기반 생성
- 동적 데이터 삽입, 다양한 포맷 지원 (PDF, HTML, Excel)
- 이메일/텔레그램 자동 발송
이 요구사항을 만족하는 자동 리포트 시스템을 설계해줘.
```

## 5번 프롬프트

```text
구현 요구사항:
1. 리포트 템플릿 시스템
2. 데이터 수집 및 처리 모듈
3. 차트 및 시각화 생성기
4. 리포트 포맷터 (PDF, HTML, Excel)
5. 자동 발송 시스템
리포트가 정기적으로 자동 생성되고 사용자에게 전송되도록 구현해줘.
```
