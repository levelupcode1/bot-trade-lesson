# 14차시: 사용자 맞춤형 자동매매 시스템

## 1번 프롬프트

```text
다양한 사용자 유형별로 자동매매 시스템 요구사항을 분석해줘
[사용자 유형별 분석]
1. 초보 투자자
   - 투자 경험: 1년 미만
   - 투자 금액: 100-500만원
   - 리스크 성향: 보수적
   - 기술적 지식: 낮음
2. 중급 투자자
   - 투자 경험: 1-3년
   - 투자 금액: 500-2000만원
   - 리스크 성향: 중간
   - 기술적 지식: 보통
3. 고급 투자자
   - 투자 경험: 3년 이상
   - 투자 금액: 2000만원 이상
   - 리스크 성향: 공격적
   - 기술적 지식: 높음
각 사용자 유형별로:
- 주요 요구사항, 우선순위
- 제약 조건, 맞춤형 기능, 사용자 인터페이스 요구사항
```

## 2번 프롬프트

```text
사용자 유형별로 맞춤형 자동매매 시스템 아키텍처를 설계해줘.
아키텍처 설계 요구사항:
1. 모듈화된 구조
   - 공통 모듈 (API, 데이터베이스, 로깅)
   - 사용자별 모듈 (전략, 인터페이스, 알림)
   - 확장 가능한 구조
2. 설정 기반 커스터마이징
   - 사용자 설정 파일
   - 동적 기능 활성화/비활성화
   - 개인화된 파라미터
3. 사용자 인터페이스
   - 기술 수준별 UI/UX
   - 모바일/웹/데스크톱 지원
   - 접근성 고려
4. 기능 선택적 제공
   - 기본 기능 (모든 사용자)
   - 고급 기능 (중급 이상)
   - 전문 기능 (고급 사용자)
```

## 3번 프롬프트

```text
위에서 설계한 맞춤형 시스템 아키텍처를 구현해줘.
구현 요구사항:
1. 사용자 프로필 관리 시스템
2. 동적 기능 로딩 시스템
3. 설정 기반 커스터마이징
4. 사용자별 인터페이스
5. 권한 관리 시스템

시스템이 사용자의 요구사항에 따라 
자동으로 맞춤형 기능을 제공하도록 구현해줘.
```

## 4번 프롬프트

```text
사용자 개인화 기능을 제공하는 시스템을 설계해줘.
개인화 기능:
1. 사용자 프로필 관리
   - 투자 성향 분석, 선호도 설정
   - 목표 설정, 제약 조건 관리
2. 맞춤형 추천 시스템
   - 전략 추천, 코인 추천
   - 설정 추천, 교육 콘텐츠 추천
3. 개인화된 대시보드
   - 관심 지표 표시, 선호하는 차트
   - 개인 알림 설정, 커스텀 위젯
4. 학습 기반 개인화
   - 사용 패턴 분석, 성과 기반 조정
   - 피드백 반영, 지속적 개선
사용자의 행동과 선호도를 학습하여 
개인화된 경험을 제공하는 시스템을 설계해줘.
```

## 5번 프롬프트

```text
위에서 설계한 개인화 시스템을 구현해줘.

구현 요구사항:
1. 사용자 행동 분석 엔진
2. 추천 시스템
3. 개인화된 대시보드
4. 학습 알고리즘
5. 피드백 수집 시스템

시스템이 사용자의 행동을 학습하여
점진적으로 개인화된 경험을 제공하도록 구현해줘.
```
